# AI Agent 安全设计要点

## 五大安全风险

| 风险 | 说明 | 防护 |
|------|------|------|
| Prompt Injection | 用户输入覆盖 System Prompt | 输入过滤 + 角色隔离 |
| 工具滥用 | LLM 调用危险工具 | 权限校验 + 人工确认 |
| 数据泄露 | Agent 返回敏感信息 | 输出过滤 + 脱敏 |
| 幻觉传播 | 错误信息回流入知识库 | 审核机制 + 置信度阈值 |
| 越权访问 | 跨租户数据访问 | RBAC + 多租户隔离 |

## 防护措施

### API Key 加密
- 存储：AES-256 加密
- 传输：HTTPS + 脱敏返回（sk-••••••••xxxx）
- 更新：检测打码值跳过更新

### SQL 注入防护（NL2SQL 场景）
- 语法校验：禁止 DROP/ALTER/TRUNCATE
- 权限控制：只读账号 + 表级权限
- 参数化：关键字段参数化拼接

### 多租户隔离
- 所有查询自动带 tenant_id
- KB/文档/会话/QA 记录按租户隔离
- 跨租户访问需显式授权
