# LangChain4j 框架重写 6 项目总结

## 框架覆盖了什么

| 组件 | 手写方式 | LangChain4j | 代码减少 |
|------|------|------|:--:|
| LLM 调用 | OkHttp 手动构建 | ChatLanguageModel | -76% |
| 工具调用 | 手动解析+反射 | @Tool + AiServices | -60% |
| 对话记忆 | ConcurrentHashMap | ChatMemory | -80% |
| 向量存储 | MySQL JSON | EmbeddingStore | -70% |

## 框架不替代什么

- 核心业务逻辑（规则引擎、编排流程）
- 安全校验（SQL 防线、权限控制）
- 多路召回融合算法
- 领域特定的 Prompt 模板设计
- 监控埋点和可观测性

## 核心认知

> 框架帮你做的是通用层（LLM 调用、工具注册、记忆管理），
> 你需要做的是领域层（业务逻辑、安全校验、质量评估）。
> 
> 会写手写版 → 理解原理
> 会用框架版 → 工程效率
> 两版对比 → 知道边界
