# Agent 记忆管理方案

## 三种记忆类型

| 类型 | 说明 | 生命周期 | 示例 |
|------|------|------|------|
| 短期记忆 | 当前对话上下文 | 单次会话 | ChatMemory |
| 长期记忆 | 跨会话的用户信息 | 持久化存储 | 用户偏好、历史对话 |
| 工作记忆 | 当前任务中间状态 | 单次任务 | 多步推理中间结果 |

## 实现方案对比

### ConcurrentHashMap（本地缓存）
- 优点：零延迟，实现简单
- 缺点：重启丢失，不可分布式
- 适用：单机 Demo、原型验证

### Redis
- 优点：分布式、持久化、TTL 自动过期
- 缺点：网络延迟、序列化开销
- 适用：生产环境会话管理

### 数据库（MySQL/PostgreSQL）
- 优点：永久存储、可查询分析
- 缺点：延迟较高
- 适用：长期记忆、历史对话归档

## 分层记忆架构

```
用户请求 → 短期记忆（当前会话）
         → 工作记忆（任务状态）
         → 长期记忆（用户画像+历史）
```
