# 多路召回融合算法

## 为什么需要多路召回

纯向量召回的局限：
- 对代码标识符不敏感（Thread.sleep ≈ 异步等待）
- 短查询语义信息少，噪声大
- 罕见模式容易被淹没
- 不可解释（为什么召回这条？）

## 三路召回融合

| 通道 | 方法 | 权重 | 优势 |
|------|------|:--:|------|
| 向量语义 | 余弦相似度 | 0.5 | 语义覆盖广 |
| 关键词 | 标识符匹配 | 0.3 | 精确匹配 |
| 标签过滤 | 知识类型 | 0.2 | 领域约束 |

## 融合算法

```java
// 1. 向量得分归一化（除以最大相似度）
normalized = score_i / max_score * weight_vector;

// 2. 关键词得分归一化（按排名倒数）
keywordScore = (1 - rank_i / total_count) * weight_keyword;

// 3. 多路命中 boost
scoreMap.merge(chunkId, score, Double::sum);
```

## 关键设计
- 不同通道用不同归一化方式（向量用分数、关键词用排名）
- 同一 chunk 被多路命中 → 得分累加
- 融合后过滤低分 + TopK 截断
