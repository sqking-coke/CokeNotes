# RAG 检索增强生成入门

## 什么是 RAG

RAG (Retrieval-Augmented Generation) = 检索 + 生成：

1. 用户提问 → Embedding 向量化
2. 向量检索 → 从知识库召回相关文档片段
3. 文档片段 + 原始问题 → 注入 LLM Prompt
4. LLM 生成带引用来源的答案

## 为什么需要 RAG

LLM 的三大局限：
- 知识截止日期（训练数据过时）
- 幻觉（编造不存在的知识）
- 无私有领域知识

RAG 通过外部知识库补充，解决这三个问题。

## RAG 核心组件

| 组件 | 作用 |
|------|------|
| 文档解析 | PDF/MD/TXT → 纯文本 |
| 文本分块 | 长文档 → 语义完整的小段 |
| Embedding | 文本 → 向量 |
| 向量数据库 | 存储 + 相似度检索 |
| Prompt 组装 | 检索结果 + 问题 → 最终 Prompt |
| 引用溯源 | 答案标注来源，可验证 |
