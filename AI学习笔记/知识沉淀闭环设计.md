# 知识沉淀闭环设计

## 问题

RAG 管道的常见缺陷：LLM 生成的好答案用完即丢。
下次有人问同样问题，又要重新检索、重新生成。

## 解决方案：知识沉淀闭环

```
QA 对话 → LLM 二次提炼 → 去重 → 审核 → 入库回流
```

### 1. LLM 提炼
不是存储 QA 原文，而是让 LLM 提取独立知识点：
- 去除对话噪音（礼貌用语、追问）
- 自包含化（脱离上下文也能读懂）
- 标注置信度 + 标签

### 2. 双重去重
- ID 去重：同一 QA 不重复提炼
- 向量去重：相似度 > 0.92 → 跳过

### 3. 人工审核
- pending → approved（参与检索）
- pending → rejected（不参与）

### 4. 回流检索
检索时 UNION ALL kb_chunks + kb_insights，标注来源类型。

### 去重阈值调优\n- 0.98：几乎逐字相同，适用同文档重复切片\n- 0.92：同义表述，适用知识沉淀去重\n- 0.85：主题相关但不重复，建议链接而非合并\n当前生产设定 0.92，通过审核通过率反馈微调。
